macOS Sierra使用GitGubPages+hexo搭建个人博客

发表于 2017-05-25 | 分类于 hexo

心路历程

  • 2017年5月23日晚，我开始折腾个人博客，使用的是GitHub Pages + Hexo静态博客框架。
    几个月前折腾过，但是没有成功，大概的原因是npm的权限问题。这次是通过修改npm的默认权限地址才成功的。
    (修改npm默认权限相关资料：https://docs.npmjs.com/getting-started/fixing-npm-permissions)
    从5月23日晚10点左右开始折腾，24日凌晨4点30分左右，成功发布第一篇博文！

  • 5月24日下午，开始折腾主题，搜索了一下，最终决定使用目前火热的主题：nexT。
    网址是：http://theme-next.iissnan.com/getting-started.html

  • 5月24日晚，由于每次发布都要加上sodu命令，强迫症的我决定重新折腾。于是备份了博客的本地文件夹（内含我的设置和喜欢的主题）。
    由于第一次是胡乱装上去的，所以忘记了具体步骤。只是本地产生了很多垃圾文件，我试图删除一些文件夹，结果把hexo删除了…命令行输入hexo -v没有显示版本信息。所以我又进入了“折腾阶段”。

  • 现在是5月25日凌晨2点，我终于再次折腾出了这个版本。在这里把大概步骤记录一下，谨防再次懵逼。

一、前面的步骤请参照博客（务必进行到“安装Hexo”这个步骤）

https://xuanwo.org/2015/03/26/hexo-intor/

二、安装HEXO

安装前请验证node、npm、git是否安装成功,均安装成功才能进行下面的步骤。对应命令如下

node node –version
npm -v
git –version

1、打开终端，cd到你博客的本地文件夹（例如：cd documents/blog），在命令行中输入以下代码

sudo npm install hexo-cli -g

2、等待步骤1完成后，在命令行中输入以下代码

sudo npm install hexo –save

步骤2如果提示：缺少pakage.json ，请依次执行以下代码
（解决这个问题的参考资料：https://github.com/npm/npm/issues/15192）

npm init -f
sudo npm install hexo –save

3、检查hexo是否安装成功（出现hexo版本信息表示安装成功）

hexo -v

4、初始化hexo

hexo init

5、安装组件（这一步我自己也没懂，随便执行了一下命令）

npm install

6、生成静态页面

hexo g

7、启动hexo本地服务器

hexo s

8、在浏览器中打开网址http://localhost:4000/ ,你将看到hexo的默认界面，至此hexo本地配置完成！

三、安装自动发布插件

1、终端cd到博客文件夹下，输入以下命令

npm install hexo-deployer-git –save

2、打开博客文件夹，打开根目录下的_config.yml文件，修改deploy为(注意替换yourusername和com.git后缀)

deploy:
type: git
repo: https://github.com/yourusername/yourusername.github.com.git
branch: master

3、配置成功后，在终端（cd到博客文件夹下）输入hexo d，如果不再需要输入github用户名和密码，说明成功了（执行命令时，可能有系统弹窗出现，选中“允许”即可）

四、发布文章、修改主题等请参照官方说明

https://hexo.io/zh-cn/docs/
